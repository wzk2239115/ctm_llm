## 测试内容

本阶段测试了针对性的 CTM 消融：

- synapse 深度；
- memory hidden 维度；
- 深层 NLM 开关；
- memory 长度；
- 输入维度和头数；
- 自条件化；
- 跨层状态。

## 为什么需要此实验

早期阶段表明默认 CTM 昂贵且不具 Transformer 竞争力。消融实验识别哪些 CTM 细节真正有帮助、哪些是不必要的，以及哪种更简单的配置应成为下一个基础。

## 结果

整个扫描中最佳 CTM 结果为：

- `s05_synapse2_mh2`：损失 `5.3612`，吞吐量 `2629 tok/s`，峰值显存 `41.6 GB`。

这优于默认 CTM 规模/tick 设置，且比大 cell 变体更实用。

其他有用结果：

- `s05_deep_nlms0`：损失 `5.4360`，吞吐量 `2795 tok/s`，显存 `37.2 GB`。
- `s05_tiny_nlm`：损失 `5.4631`，吞吐量 `2703 tok/s`，显存 `40.6 GB`。
- `s05_no_selfcond`：损失 `5.4551`，说明自条件化在此设置中并非明显必需。
- `s05_no_cross_state`：损失 `5.5350`，比 `no_selfcond` 更差，表明跨层状态比自条件化更重要。
- `s05_short_memory`：损失 `5.5233`，比基线 memory 长度更差，说明过短的 memory 有害。

## 结论

最佳 CTM 方向不是更大的 cell 或更多的 tick，而是围绕 `s05_synapse2_mh2` 的更干净、更廉价的 CTM 核心。

## 关联比较

结合以下内容使用：

- `s01_baseline_scale.md`：这是与 Transformer 对比的最佳 CTM 候选；
- `s02_tick_dynamics.md`：将此基础与 tick1/tick2/tick3 扫描配对；
- `s03_elf.md`：在此基础上添加短 horizon ELF；
- `s04_cells_sparsity.md`：将此基础与真正的稀疏执行结合，而非仅 top-k 掩码。

## 优势或劣势诊断

`s05_synapse2_mh2` 可能更好的原因：

- 减少了不必要的 synapse 复杂度。
- 通过简化 cell 动态可能改善优化。
- 保留了足够的 memory 结构，而没有过深的内部非线性模块。

其他消融落后的原因：

- 移除跨层状态削弱了信息流。
- 短 memory 减少了时序上下文。
- 浅 synapse 深度可能欠拟合，相比于更平衡的 synapse2 设置。

## 后续

以 `s05_synapse2_mh2` 作为下一个 CTM 基础，运行聚焦的第二轮扫描：

1. tick 数：1、2、3、4；
2. ELF horizon：无、2、4；
3. 带实际计算跳过的动态停机；
4. 带块级跳过的真正稀疏 cell；
5. 等显存和等墙钟时间下的匹配 Transformer 对照。

下一轮成功标准：

- CTM 损失在相同训练预算下低于 `5.30`；
- 峰值显存低于 `45 GB`；
- 吞吐量高于 `3000 tok/s`；
- 有证据表明额外 tick 改善了特定困难样本，而非均匀改善所有 token。
